# NumericSwift Parity Fixtures

This directory contains frozen artifacts that anchor the Phase 0 parity baseline
for the unified-numeric-pipeline milestone (task-master tag: `unified-pipeline`).

## Files

### `LegacySnapshot.json`

Frozen snapshot of the THREE legacy evaluator surfaces:

1. `MathExpr.evaluate` — real scalar path (`MathExpr.swift:150`)
2. `MathExpr.evaluateComplex` — complex scalar path (`MathExpr.swift:311`)
3. `LinAlg` direct function/operator calls — matrix/linalg surface (`LinAlg.swift`)

The snapshot was generated by `LegacySnapshotGeneratorTests/testGenerateSnapshot`
using `NUMERICSWIFT_REGENERATE_SNAPSHOT=1 swift test --filter LegacySnapshotGeneratorTests`.

### `codesize-baseline.json`

Phase 0 release-build binary size baseline. Records the `swift build -c release`
module object size before any unification work. Later phases compare against this
to catch accidental code-size regressions.

## Value kinds covered

| Kind | Description |
|------|-------------|
| `scalar` | `Double` result from real-scalar path |
| `complex` | `Complex(re:im:)` from complex-scalar path |
| `matrix` | `LinAlg.Matrix` (row-major) from LinAlg path |
| `complexMatrix` | `LinAlg.ComplexMatrix` (row-major re+im) from LinAlg path |
| `nilResult` | Optional return resolved to nil (e.g. inv of singular matrix) |
| `error` | Thrown or expected-trap error, stored by category |

IEEE-754 special values (NaN, +inf, -inf, +0, -0) are encoded as `UInt64` bit
patterns so they survive JSON round-trips exactly. NaN != NaN under standard
IEEE 754 comparison; parity tests use `.isNaN` / `.bitPattern` comparisons.

## Error categories

- `dimensionMismatch` — Group-A shape-mismatch (precondition in legacy; throw in unified)
- `divisionByZero` — Scalar or matrix division by zero
- `notSquare` — Group-B non-square input to trace/det/inv/expm/logm/sqrtm/cdet/cinv
- `invalidArguments` — Invalid or unsupported argument
- `parseError` — Expression parse failure
- `undefinedVariable` — Undefined variable reference

## The no-regenerate-from-unified rule

**NEVER** regenerate `LegacySnapshot.json` from unified-pipeline code. Doing so
produces a tautologically-true fixture (the unified evaluator always matches
itself) that cannot catch regressions. This is the "vacuous-gate bug."

`LegacySnapshot.json` is the source of truth. The unified evaluator is correct
if and only if it produces outputs that match this snapshot.

## Legitimate regeneration procedure

`LegacySnapshot.json` MAY be regenerated ONLY when the corpus itself changes
(new entries added, semantics clarified) using:

```bash
NUMERICSWIFT_REGENERATE_SNAPSHOT=1 swift test \
    --filter NumericSwiftTests.LegacySnapshotGeneratorTests/testGenerateSnapshot
```

Review the diff before committing. The generator must use ONLY the legacy
evaluators (`MathExpr.evaluate`, `MathExpr.evaluateComplex`, `LinAlg.*`).
